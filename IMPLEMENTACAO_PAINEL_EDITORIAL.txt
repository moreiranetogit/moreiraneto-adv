🧪 COMO TESTAR
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. PREPARAÇÃO
   a) Executar schema SQL no Supabase:
      → Copiar conteúdo de: supabase/artigos-schema.sql
      → Colar no SQL Editor do Supabase
      → Executar (cria todas as tabelas)
      → Dados iniciais carregam automaticamente

2. TESTAR DASHBOARD
   a) npm run dev
   b) Acesse: http://localhost:3000/admin/artigos
   c) Deve ver: "Nenhuma notícia encontrada" ou lista vazia
   d) Clique "+ Adicionar"
   e) Preencha: Título, Descrição, Fonte, Categoria
   f) Clique "Adicionar"
   g) Notícia aparece na lista

3. TESTAR REVISÃO
   a) Clique "Revisar" em uma notícia
   b) Vê: Notícia original + campo de análise vazio
   c) Edite o campo de análise
   d) Clique "Salvar Rascunho"
   e) Volte ao dashboard e veja status "pendente"

4. TESTAR PUBLICAÇÃO
   a) Na página de revisão, clique "Aprovar e Publicar"
   b) API retorna sucesso
   c) Status deve mudar para "publicado"
   d) Nota: /despacho ainda não existe (Fase 2)

5. TESTAR FILTROS
   a) Dashboard: busque por título
   b) Filtre por categoria
   c) Filtre por status
   d) Combine filtros
